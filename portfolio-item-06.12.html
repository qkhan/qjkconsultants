Project Description


  1. Web Spiders are network applications which traverse the Web, accumulating statistics about the content found.

  2. Create a queue of URLs to be searched beginning with one or more known URLs. Pull a URL out of the queue and fetch the Hypertext Markup Language (HTML) page which can be found at that location.

  3. Scan the HTML page looking for new-found hyperlinks. Add the URLs for any hyperlinks found to the URL queue

Technical Detail


  1. Using the above technique, I have implemented new databases for AUSTLII Search Engine. I have designed & develop Web Spiders in PERL, parsing HTML/XML files from various court web sites / repositories.

  2. Applying complex Regular Expression to extract key information like citations, authors, titles, etc.

  3. Parsing Unstructured HTML / PDF pages; filter information to create structured Vanilla style HTML pages.

  4. Maintain current databases, implement new algorithms, correct issues like change of web site structure, missing key data information, duplication, etc

  5. Extensive use of CPAN modules, to name few like HTML::TreeBuilder, XML::Lite etc

  6. Coding, reviewing, modifying, testing/debugging individual applications and integrating sub systems to work as one big system
