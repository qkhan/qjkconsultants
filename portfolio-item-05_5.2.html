Project Definition


    Designed and developed “Payment Batch Processing” application, to enable automation of the future periodic payments for each claim.

  1. The claim officer creates separate batches of claims with date range and run them accordingly. The implementation of the batch processing ensures enhanced automation of the claim processing life cycle from calculating the payments, preparing financial extractions, updating work flow tasks, generate dynamic letters, process payments and create a repository for the claim including relevant documents for house-keeping.


  2. The user can access the batch processing system from the main page of the Claim system.


  3. The first pages allows user to add new batch or use the existing batch to change dates.


  4. Add New Batch button will allow user to create new batch


  5. This will create new batch with details as shown below:


  6. User is allowed to change the details of the Batch, delete the batch or view claims within that batch.


  7. View/Add Claims button will take the user to the screen below:


  8. The claims officer will then be able to add his/her assigned claims or could add claims of another claims officer


  9. The user could select multiple claims officers or All-Claims-Officers to view claims and add to the batch list


  10. The user will select claims from the list shown and click “Add Batch Entries”


  11. The screen will be updated as below, showing claims added but not processed


  12. If the user selects “-“ from the combo box for the option “View other Users”, it will filtered the claims list to the ones, selected only.


  13. User can repeat the process to add more claims to the batch list.


  14. At this stage, the user can press the button “Run Batch Entries” and it will run payments for the claims added.


  15. The processing run will check the payments conditions and highlight issues with the claims payment or three asterisk *** complete to advise, the batch processing has been successful.


  16. The link “Check Status of the Batch” above will take the user to the batch list with the result of the batch processing


  17. The ones processed will be ticked off.

  18. Claims officer at this stage will examine the error and fix it accordingly


  19. The claim ID: BLU100433 which is a hyperlink, will take the claims officer to the benefit payment screen


  20. The user fix the errors for the batch, which states Batch Payment Start date, should be after “Benefit Commencement Date” and payment end date should be before “Estimated Return to Normal Duties”.


  21. The claims officer fixes all the errors and then run batch with the group of claims having same benefit payment start and the end date.




Key System Functionality

  1. The system provides all kinds of insurance claims
  2. The system provides the flexible connectivity essential to effective claims management.
  3. It reduces time spent doing routine administrative tasks.
